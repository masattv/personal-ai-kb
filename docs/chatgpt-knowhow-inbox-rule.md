# ChatGPT会話で出たノウハウを inbox に残すルール

- 作成日: 2026-06-08
- 元Issue: #58
- 分類: AI活用 / ナレッジ運用 / inbox運用

## 目的

ChatGPTとの会話で出た実用的なノウハウを、後から検索・再利用できるように `inbox` に記録する。

単なる雑談や一時的な感想ではなく、手順化・資料化・再利用できる内容を残す。

## 記録対象

- 手順化できるノウハウ
- 設定方法やトラブルシュート
- プロンプトやテンプレート
- 開発、AI活用、投資、転職、生活改善などで再利用できる知見
- 後から復習したい内容
- 後で資料化・Markdown化したい内容

## 記録しないもの

- その場限りの感想
- 文脈がない断片メモ
- 個人情報、社外秘、業務コードなど外部サービスに貼るべきでない情報
- 既に `knowledge/` や `docs/` に反映済みの重複内容

## 記録フォーマット

```md
## タイトル

- 日付: YYYY-MM-DD
- 出典: ChatGPT会話
- 分類: AI活用 / 開発 / 生活 / 投資 / 転職 / 学習 など

### 要点

- 

### 手順・具体例

- 

### 次に使うときの注意

- 
```

## 運用メモ

会話中に「これは再利用できる」と判断できるノウハウは、まず GitHub Issue の `inbox` に残す。

その後、内容が十分に具体的なら `knowledge-candidate` として分類し、反映先が明確になった時点で `routed-knowledge` にする。Markdownへ反映したら `reflected` を付ける。
