# 概要

GitHubを個人ナレッジの本体として使い、スマホから雑に入力し、AIで整理し、Claude CodeやMCPで再利用する。

# 基本方針

- Issueは雑に入れる場所
- daily issue commentは日々のログ
- blog/は日次Markdown
- knowledge/は整理済みナレッジ
- Claude Codeは整理・変換・レビュー係
- MCPは外部ツール連携・高度操作用

# 次にやること

- [ ] GitHub repoを作成する
- [ ] ZIPを展開して初回コミットする
- [ ] PERSONAL_AI_KB_TOKENをActions Secretsに登録する

- [ ] Export Daily Issueを手動実行する
